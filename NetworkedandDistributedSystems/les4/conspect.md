## Практика 4

Пусть у нас у каждого компьютера есть свое локальное время, у сервера - свое, у клиента - свое

```c
#include <stdlib.h>

//инициализация
void srandom(unsigned int seed) //выбираем seed равный рангу
//выдает псевдослучайное число в диапазоне от 0 до RAND_MAX
long int random(void)
// пользуемся
// Для получения целого случайного числа в диапазоне от a до b включительно нужно использовать значение a + random()%(b-a+1)
```

Процесс с рангом 0 – сервер времени, все остальные клиенты.

В начале работы каждый процесс инициализирует генератор случайных чисел и случайным образом устанавливает свои локальные часы. 

Каждый клиент один раз обращается к серверу времени для синхронизации часов. Сервер времени принимает запросы в произвольном порядке. 

### Работа сервера:
1) Инициализируем генератор случайных чисел рангом процесса.
2) Присваиваем часам UTC случайное значение от 0 до 20.
3) Начало цикла по количеству клиентов.
4) Принимаем сообщение от произвольного клиента.
5) Прибавляем к часам UTC случайное значение от 0 до 20. Это будет время UTC приема сообщения. 
6) Имитируем время обработки сообщения случайной величиной от 0 до 3. Это будет величина d.
7) Прибавляем к часам UTC значение d. Это будет время T_utc отправки сообщения. 
8) Отправляем клиенту сообщение, содержащее T_utc и  d.
9) Конец цикла по количеству клиентов.

### Работа клиента:
1) Инициализируем генератор случайных чисел рангом процесса.
2) Присваиваем локальным часам значение от 0 до 20. Это будет время T0. 
3) Отправляем запрос серверу.
4) Ждем ответа от сервера, читаем TUTC и  d.
5) Прибавляем к значению локальных часов случайное значение от 5 до 10, имитируя время на прохождение сообщений и работу 6) сервера. Полученное значение – время T1.
6) Вычисляем необходимую поправку для локальных часов и немедленно устанавливаем правильное время. 
7) Значение (T1 - T0 – d )/2 округляем до целого сверху.

### Выдача от сервера и клиентов
Сервер печатает, от кого принял сообщение, в какое время, а также кому отправил сообщение, в какое время и значение d.

Клиент печатает значение времен T0, T1, содержимое принятого сообщения (TUTC и d) и исправленное значение локального времени.
