## Практика 2

Есть 0 до 2, разбили на кучу отрезочков, отправили серверам, считают одновременно и тд

Поскольку мы все работаем на кластере, может быть такое что сервера могут быть не реализованы

Иная трактовка: 1 сервер, клиент нарезает отрезки на зону ответственности, это получает сервер, считает, присылает datagramm-у с частичными суммами, просуммировали, получили ответ

Количество разбиений, определяющих шаг по гор. прямой по оси Х - N отрезков

Считаем зоны ответственности, разбиваем на k одинаковых частей

Отправляем datagram-у 

С клавиатуры вводятся N, k

в цикле посылаем k datagram серверу

Каждая datagram содержит указания, чтобы сервер проделал работу

Datagram должна состоять из 3ех чисел: N, ibeg, iend - номера начала и конца зон ответственности (все int)

Отрезки нумеруем с нуля, всего номеров от 0 до N

Сервер ждет datagram, принимает, считает Si, отправляет Si

Не преобразовываем числа в сетевой порядок байт, не надо этого делать

Советы: 
Начать с клиента, мы сможем отлаживать через идеальный сервер
После рабочего клиента мы пишем свой сервер, используя отлаженного клиента можем отлаживать сервер

cannot find port, ctrl+z не убирает сервер, используем лишь ctrl+c для выруба сервера
